2025-11-17 调试虚拟环境遇到的问题

Created2025-11-17|Updated2025-11-17
|Word Count:590|Post Views:

最近想要开始学习一下pytorch,发现各个pytorch版本对应的cuda版本,包版本都不一样.就想着装一个新的python虚拟环境来着,结果遇到了各种各样让人头痛的问题,经过无数次的搜索和浏览最后解决了,所以在这里分享一下我遇到的问题,也防止后人踩坑.

1.安装虚拟环境

由于吃过anaconda臃肿体积,启动巨慢的苦,果断换成没用过的venv虚拟环境,结果运行python -m venv .venv之后没在根目录里发现这个文件夹.运行.\.venv\Scripts\activate激活后才看到.

一开始我以为这样弄的虚拟环境要在vscode里专门改python解释器,调试来调试去,问了ai无数遍,结果发现vscode可以自动识别使用当前目录里的python解释器,直接在当前目录打开vscode就能运行,自己成傻子了.

2.导入tensorflow


一开始我想着去官网找导入命令,但界面里的无用信息太多了,找了一会就不想找了.


后来用了菜鸟教程里的导入,结果导入成cpu的了,最后还是用gpt大法找到了正确的导入代码.

pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu130
–安装gpu版本的torch

3.code runner bug

当我用code runner运行虚拟环境里的python文件时,文档路径有空格导致python解释器报错.

又想起来以前用code runner 因为编码出现各种各样的问题,果断改用python解释器单独运行,才解决了这个报错问题.

以后遇到各种各样的问题我可能都会传上来,因为这些琐碎的操作过了几个月就全忘了,如果哪一天用得上的话又得重复去各个网站找,烦死人,最近我就在调整文章无法显示封面图片的问题,等我解决了之后就会传上来.
